# 本地部署说明文档

## 1. 文档信息

- 项目名称：演示系统
- 文档类型：本地部署说明
- 所属仓库：`yanshi_be`
- 当前状态：定稿初版
- 创建日期：2026-06-21
- 最后更新：2026-06-21

## 2. 项目说明

当前项目包含两个代码仓库：

- 前端仓库：`luxuyuan-candi/yanshi_fe`
- 后端仓库：`luxuyuan-candi/yanshi_be`

当前网站规划包含两个页面：

1. 登录页
2. 首页

本文档用于说明项目在本地环境下的运行部署方式，作为开发、联调和演示参考。

## 3. 已确认部署要求

当前已确认的技术方案如下：

1. 后端使用 `Python + FastAPI`
2. 采用本地部署
3. 前端只需要 `HTML` 文件
4. 数据库使用本地 `SQLite`

## 4. 部署目标

部署完成后，应满足以下目标：

1. 本地可正常启动后端 `FastAPI` 服务。
2. 本地可直接打开前端 `HTML` 页面。
3. 前后端可正常联调，登录和首页功能可闭环使用。
4. 本地 `SQLite` 数据库可正常读写。
5. 项目具备可重复执行的本地启动流程。

## 5. 本地部署架构
当前推荐采用最简本地运行架构：

- 前端使用静态 `HTML` 文件。
- 后端使用 `FastAPI` 本地启动。
- 数据库使用项目目录下的 `SQLite` 文件。
- 前端通过浏览器直接访问本地页面，或通过本地静态服务访问页面。
- 前端调用本地 `FastAPI` 接口完成登录和首页数据获取。

推荐访问链路如下：

1. 用户在本地浏览器中打开前端页面。
2. 前端页面向本地 `FastAPI` 服务发起接口请求。
3. `FastAPI` 服务读取或写入本地 `SQLite` 数据库。
4. 后端返回数据后，前端完成页面展示。

## 9. 部署方式说明

### 9.1 前端本地部署方式

前端仓库 `yanshi_fe` 只需要提供静态 `HTML` 页面，推荐方式如下：

1. 编写 `login.html` 和 `index.html`。
2. 如有样式和脚本，可按需拆分到 `assets/css` 和 `assets/js`。
3. 浏览器直接打开页面文件，或使用本地静态服务启动页面。
4. 页面内通过 `fetch` 调用后端接口。

推荐说明：

- 如直接通过 `file://` 打开页面，可能受浏览器跨域策略影响。
- 更稳妥的方式是通过一个轻量本地静态服务访问 `HTML` 文件。
- 若使用 VS Code Live Server，则可直接在本地端口访问页面。

### 9.2 后端本地部署方式

后端仓库 `yanshi_be` 使用 `FastAPI`，推荐方式如下：

1. 创建 Python 虚拟环境。
2. 安装 `requirements.txt` 依赖。
3. 配置本地环境变量。
4. 初始化 `SQLite` 数据库。
5. 使用 `uvicorn` 启动 `FastAPI` 服务。

推荐启动命令示例：

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

说明：

- `--reload` 适合本地开发环境。
- 如目录结构不同，需要调整 `app.main:app` 路径。

### 9.3 数据库本地部署方式

数据库使用本地 `SQLite`，推荐方式如下：

1. 在后端项目中创建 `data` 目录。
2. 首次启动时自动创建数据库文件，或由初始化脚本创建。
3. 数据库连接串示例：

```text
sqlite:///./data/app.db
```

说明：

- `SQLite` 适合当前轻量项目和本地演示场景。
- 无需单独安装数据库服务。

## 10. 部署前准备项

部署前建议先确认以下内容：

1. 前端和后端仓库代码已拉取到目标服务器。
2. 本地已安装 `Python` 和 `pip`。
3. 后端依赖文件已准备好。
4. 前端页面已明确接口调用地址。
5. 本地端口未被其他程序占用。
6. 数据库目录具备读写权限。

## 11. 配置项建议

### 11.1 前端配置建议

建议前端按环境区分以下配置：

| 配置项 | 说明 |
| --- | --- |
| `API_BASE_URL` | 后端接口基础地址，例如 `http://127.0.0.1:8000` |
| `APP_TITLE` | 系统名称 |

### 11.2 后端配置建议

建议后端按环境区分以下配置：

| 配置项 | 说明 |
| --- | --- |
| `PORT` | 后端服务端口 |
| `SQLITE_DB_PATH` | SQLite 数据库文件路径 |
| `JWT_SECRET` | 登录态签名密钥 |
| `CORS_ORIGIN` | 跨域来源配置 |
| `LOG_LEVEL` | 日志级别 |

### 11.3 配置管理原则

- 本地环境建议使用 `.env` 或配置文件管理。
- 敏感信息不直接写死在业务代码中。
- 即使是本地项目，也建议保留最基本的配置分层。

## 12. 本地启动步骤

### 12.1 后端启动步骤

1. 进入后端仓库目录。
2. 创建虚拟环境。
3. 激活虚拟环境。
4. 安装依赖。
5. 启动 `FastAPI` 服务。

命令示例：

```bash
cd yanshi_be
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 12.2 前端启动步骤

1. 进入前端仓库目录。
2. 准备 `login.html` 和 `index.html` 文件。
3. 使用浏览器直接打开，或通过本地静态服务启动。

如使用 Python 自带静态服务，可参考：

```bash
cd yanshi_fe
python3 -m http.server 5500
```

启动后可访问：

- 登录页：`http://127.0.0.1:5500/login.html`
- 首页：`http://127.0.0.1:5500/index.html`

### 12.3 跨域建议

由于前端和后端本地端口可能不同，建议后端开启跨域支持。

建议允许来源：

- `http://127.0.0.1:5500`
- `http://localhost:5500`

## 13. 标准本地部署流程

1. 拉取前后端仓库代码。
2. 准备前端静态页面。
3. 配置后端依赖与运行环境。
4. 初始化本地 `SQLite` 数据库。
5. 启动 `FastAPI` 服务。
6. 启动本地前端静态服务。
7. 打开登录页验证接口是否可用。
8. 登录后进入首页并验证页面功能。

## 14. 部署后验证清单

部署完成后，至少验证以下内容：

1. 浏览器可正常打开登录页。
2. 用户可正常登录。
3. 登录后可正常进入首页。
4. 首页刷新后登录态仍符合预期。
5. 退出登录功能可正常使用。
6. 接口请求无明显跨域问题。
7. `SQLite` 数据文件可正常生成或读写。
8. 后端控制台无严重报错。

## 16. 常见问题建议

1. 页面能打开但接口报错：
   检查后端是否启动，接口地址是否写对。
2. 接口返回跨域错误：
   检查 `FastAPI` 是否已配置 `CORS`。
3. 数据未保存：
   检查 `SQLite` 路径和目录权限。
4. 登录后无法进入首页：
   检查前端登录态存储逻辑和接口返回结构。
